# 📘 SOC L1 SIEM Lab – Project Overview

---

## 📌 Overview

This project showcases the design and implementation of a **Security Information and Event Management (SIEM)** solution using **Splunk** in a simulated **SOC (Security Operations Center) Level 1 environment**.

It replicates real-world SOC workflows, including **log ingestion, detection engineering, threat analysis, and incident investigation**.

---

## 🎯 Key Objectives

* Monitor and analyze authentication logs
* Detect suspicious or anomalous activity
* Develop detection rules using SPL (Search Processing Language)
* Investigate potential security incidents
* Map detections to the MITRE ATT&CK framework
* Document findings in a structured format

---

## 🛠️ Tools & Technologies

* Splunk Enterprise
* Ubuntu Linux
* Oracle VirtualBox
* Linux authentication logs (`/var/log/auth.log`)
* SPL (Search Processing Language)

---

## ⚙️ Environment Overview

A virtual SOC lab was built using **Ubuntu Linux** hosted on **VirtualBox**, with **Splunk Enterprise** configured for centralized log collection and analysis.

Authentication logs from the system were ingested into Splunk to simulate real-world monitoring scenarios.

---

## 📥 Log Ingestion

* Configured Splunk to ingest Linux authentication logs
* Validated data ingestion and field extraction
* Enabled visibility into login activity and system events

---

## 🔍 Detection & Analysis

* Identified failed login attempts and brute-force patterns
* Analyzed authentication behavior
* Detected anomalous access activity
* Investigated events using SPL queries

---

## 🧠 Skills Demonstrated

* SIEM Monitoring & Operations
* Log Analysis & Event Correlation
* Detection Engineering (SPL)
* Threat Investigation
* MITRE ATT&CK Mapping
* Technical Documentation

---

## 📊 Outcome

* Successfully ingested and analyzed log data
